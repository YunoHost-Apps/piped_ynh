# Le problème

YouTube a une politique de confidentialité extrêmement intrusive qui repose sur l’utilisation des données des utilisateurs de manière peu éthique. Vous lui confiez beaucoup de données — idées, goûts musicaux, opinions politiques, et bien plus que vous ne le pensez.

En utilisant Piped, vous pouvez regarder et écouter du contenu librement sans craindre qu’on surveille tout ce que vous faites.

## Fonctionnalités

**Fonctionnalités utilisateur**

-   [x] Sans publicité
-   [x] Sans traçage
-   [x] Léger côté serveur et client
-   [x] Défilement infini
-   [x] Thèmes clair/sombre
-   [x] Connexion
-   [x] Flux
-   [x] Listes de lecture
-   [x] Intégration avec [SponsorBlock](https://github.com/ajayyy/SponsorBlock)
-   [x] Intégration avec [LBRY](https://lbry.com/) pour le streaming
-   [x] Intégration avec [Return YouTube Dislike](https://returnyoutubedislike.com/) via [RYD-Proxy](https://github.com/TeamPiped/RYD-Proxy)
-   [x] Prise en charge de la 4K
-   [x] Aucune connexion aux serveurs de Google
-   [x] Lecture audio seule
-   [x] Support PWA
-   [x] Préférences enregistrées localement
-   [x] [Disponible en de nombreuses langues](https://github.com/TeamPiped/Piped/tree/master/src/locales), grâce à [nos traducteurs](https://hosted.weblate.org/projects/piped/frontend/)
-   [x] Support des vidéos intégrées
-   [x] Pas de restriction d’âge
-   [x] Contourne les restrictions géographiques si possible via un réseau fédéré

**Fonctionnalités techniques**

-   [x] Répartition de charge multi‑région
-   [x] Performant par conception, prévu pour gérer des milliers d’utilisateurs simultanés
-   [x] N’utilise pas les API officielles de YouTube
-   [x] Utilise [NewPipeExtractor](https://github.com/TeamNewPipe/NewPipeExtractor) pour extraire les informations
-   [x] [API JSON publique](https://docs.piped.video/docs/api-documentation/)
-   [x] Protocole fédéré sur Matrix pour permettre aux instances de collaborer entre elles
